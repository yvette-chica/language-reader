## Concepts Covered

### Tailwind v4 setup

Tailwind v4 is much simpler than v3 вЂ” no `tailwind.config.js` needed. Just install the Vite plugin and add one line to your CSS:

```js
// vite.config.js
import tailwindcss from '@tailwindcss/vite'
export default defineConfig({ plugins: [react(), tailwindcss()] })
```

```css
/* index.css */
@import "tailwindcss";
```

Tailwind scans your JSX files automatically and generates only the CSS classes you actually use.

---

### Tailwind utility classes

Instead of writing CSS, you apply small utility classes directly in JSX:

```jsx
<div className="min-h-screen flex items-center justify-center bg-gray-50">
```

- `min-h-screen` в†’ `min-height: 100vh`
- `flex` в†’ `display: flex`
- `items-center` в†’ `align-items: center`
- `justify-center` в†’ `justify-content: center`
- `bg-gray-50` в†’ a very light gray background

### React Router

React Router lets each screen have its own URL. The setup:

```jsx
// main.jsx вЂ” wrap the whole app
<BrowserRouter><App /></BrowserRouter>

// App.jsx вЂ” define routes
<Routes>
  <Route path="/login" element={<LoginPage />} />
  <Route path="/" element={<CoursesPage />} />
</Routes>
```

`useNavigate()` вЂ” hook for programmatic navigation (e.g. after login):
```js
const navigate = useNavigate()
navigate('/')
```

`useParams()` вЂ” hook to read URL parameters:
```js
const { courseId } = useParams() // from /courses/:courseId
```

### Protected routes

A wrapper component that checks for a token before rendering a page вЂ” redirects to `/login` if not logged in:

```jsx
function ProtectedRoute({ children }) {
  if (!isLoggedIn()) return <Navigate to="/login" replace />
  return children
}

// Usage
<Route path="/" element={<ProtectedRoute><CoursesPage /></ProtectedRoute>} />
```

### Vite dev proxy

During development, the React app runs on port 5173 and the API on 3001. Without a proxy, the browser would block cross-origin requests. The proxy forwards `/api/...` requests transparently:

```js
// vite.config.js
server: {
  proxy: {
    '/api': 'http://localhost:3001',
  },
},
```

So `fetch('/api/courses')` in React automatically hits `http://localhost:3001/api/courses`.

### `useEffect` for data fetching

`useEffect` runs after the component renders. Used to fetch data from the API on page load:

```js
useEffect(() => {
  api.get('/courses')
    .then(data => setCourses(data))
    .catch(err => setError(err.message))
    .finally(() => setLoading(false))
}, []) // empty array = run once on mount
```

### Sentence splitting with regex lookbehind/lookahead

The manual transcript route splits text into sentences without a NLP library:

```js
text.split(/(?<=[.!?])\s+(?=[A-ZГЃГ‰ГЌГ“Г–ЕђГљГњЕ°])/)
```

- `(?<=[.!?])` вЂ” lookbehind: the split point must follow a sentence-ending punctuation
- `\s+` вЂ” one or more whitespace characters (the actual split point)
- `(?=[A-Z...])` вЂ” lookahead: the next word must start with a capital letter

This keeps punctuation attached to the sentence that ends with it, and handles German and Hungarian capital accented letters.

## Troubleshooting

**`faster_whisper` module not found** вЂ” Node spawned the system Python instead of the venv Python. Fixed by using the full venv path:
```js
const PYTHON = join(__dirname, '../.venv/bin/python')
```

**Whisper segments had null timestamps** вЂ” the Python script outputs `start_time`/`end_time` but the JS was reading `seg.start`/`seg.end`. Fixed the key names.

**`set -e` killed the seed script silently** вЂ” `grep` returns exit code 1 when it finds no match, which kills the script under `set -e`. Fixed with `|| true` after each `grep` call.

**Wrong port in seed script** вЂ” server runs on 3001, script had 3000.

**Vite 8 requires Node 20.19+** вЂ” resolved by upgrading Node system-wide via pacman (removed `balena-etcher` which depended on `nodejs-lts-jod`, then installed `nodejs`).

**rolldown native binding missing** вЂ” npm bug with optional dependencies. Fixed by deleting `node_modules` and `package-lock.json` and running `npm install` fresh.
