## Concepts Covered

### `useRef` for DOM elements

`useRef` gives you a stable reference to a DOM node (or any value) that persists across renders without causing a re-render when it changes. Used here to access the `<audio>` element directly:

```jsx
const audioRef = useRef(null)
// ...
<audio ref={audioRef} ... />

// Later, read the current playback time:
const t = audioRef.current?.currentTime
```

The `?.` is optional chaining вЂ” if `audioRef.current` is null (e.g. the component hasn't mounted yet), it safely returns `undefined` instead of throwing.

---

### The `timeupdate` audio event

The HTML `<audio>` element fires `timeupdate` several times per second while playing. Used to track which transcript segment is currently being spoken:

```jsx
<audio onTimeUpdate={handleTimeUpdate} ... />
```

```js
function handleTimeUpdate() {
  const t = audioRef.current?.currentTime
  const active = segments.find(
    seg => seg.start_time != null && t >= seg.start_time && t < seg.end_time
  )
  setActiveSegmentId(active?.id ?? null)
}
```

---

### Avoiding unnecessary re-renders

`timeupdate` fires many times per second. Calling `setState` on every event вЂ” even when the value hasn't changed вЂ” would cause many wasted re-renders. The fix: only update state when the active segment actually changes:

```js
setActiveSegmentId(prev => prev === id ? prev : id)
```

This pattern (comparing previous state before updating) is useful any time you have a high-frequency event driving state.

---

### Layout components

A layout component is a wrapper that adds shared UI (like a nav bar) around page content without each page needing to import it individually:

```jsx
function ProtectedLayout({ children }) {
  if (!isLoggedIn()) return <Navigate to="/login" replace />
  return (
    <>
      <Nav />
      {children}
    </>
  )
}
```

`children` is a special React prop вЂ” whatever you put between the opening and closing tags of a component. So `<ProtectedLayout><CoursesPage /></ProtectedLayout>` passes `<CoursesPage />` as `children`.

---

### When to extract a component

We discussed whether to split `Segment` and other JSX into their own files. The conclusion: extract when a component is **reused in multiple places** or when a file becomes **too long to navigate**. Neither applied here вЂ” `Segment` is only used in `LessonPage` and the files are all under ~200 lines. Premature extraction adds file overhead and makes components harder to understand in isolation.
