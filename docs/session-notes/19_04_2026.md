## Concepts Covered

### Saving an auth token as a shell variable

```bash
TOKEN=$(curl -s -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"yourpassword"}' \
  | jq -r '.token')

# Then use it
curl http://localhost:3001/api/words -H "Authorization: Bearer $TOKEN"
```

The variable lives for the duration of the terminal session.

---

### `cut -d'"' -f4`

`cut` splits a string by a delimiter and returns the Nth field. Given `"token":"eyJhbG..."`, splitting on `"` gives:

| field | value |
|-------|-------|
| 1 | *(empty)* |
| 2 | `token` |
| 3 | `:` |
| 4 | `eyJhbG...` |

So `-f4` extracts just the token value. `jq -r '.token'` does the same thing more cleanly if `jq` is available.

---

### JSON columns in SQLite

SQLite doesn't have a native JSON type but you can store JSON as a `TEXT` column. The server serialises to a string on write and parses it back to an object on read вЂ” the frontend never sees raw JSON strings.

```js
// Writing
attributes ? JSON.stringify(attributes) : null

// Reading
attributes: row.attributes ? JSON.parse(row.attributes) : null
```

Useful when you have flexible, language-specific metadata that doesn't need to be queried or indexed.

---

### SQL `COALESCE`

`COALESCE(a, b)` returns the first non-null value. Used in PATCH to only update fields that were provided:

```sql
UPDATE words SET
  status = COALESCE(?, status),
  translation = COALESCE(?, translation)
WHERE id = ?
```

If you pass `null` for `status`, the existing value is kept. If you pass a new value, it's used.

---

### HTTP 204 No Content

The convention for DELETE responses вЂ” success, but no body to return. In Express:

```js
res.status(204).send();
```

---

### Two endpoints vs one endpoint with a `method` field

When two operations act on the same resource but are fundamentally different in what they do and what data they need, separate endpoints are cleaner:

```
POST .../transcript/whisper   вЂ” triggers audio transcription
POST .../transcript/manual    вЂ” accepts pasted text
```

vs a single `POST .../transcript` with `{ "method": "whisper" }` вЂ” which works but leads to a large if/else in the route handler.

### Local vs cloud Whisper

| | Local (faster-whisper) | Cloud (OpenAI API) |
|---|---|---|
| Cost | Free | $0.006/min of audio |
| Speed | Depends on hardware | Fast |
| Setup | pip install + model download | API key |
| Privacy | Audio never leaves machine | Sent to OpenAI |

For an i5 8th gen (CPU only), the `small` model is the right balance вЂ” accurate enough for clear speech, fast enough to be practical. `medium` would be noticeably slower.

The `int8` compute type reduces memory usage with minimal accuracy loss вЂ” a good default for CPU.

### Child processes in Node (`spawn`)

Node can run external programs as child processes. `spawn` is the right tool when you want to stream the output rather than buffer it all in memory:

```js
import { spawn } from 'child_process';

const python = spawn('python', ['script.py', arg]);

let output = '';
python.stdout.on('data', (data) => { output += data; });

python.on('close', (code) => {
  if (code !== 0) { /* handle error */ }
  const result = JSON.parse(output);
});
```

The Python script prints JSON to stdout; Node reads it back and parses it.

---

### yt-dlp

A command-line tool for downloading audio/video from YouTube and many other sites. Used here to extract audio before passing it to Whisper:

```bash
yt-dlp -x --audio-format mp3 -o /tmp/output.mp3 "https://youtube.com/..."
```

`-x` means extract audio only (don't keep the video).

---

### `pip install` vs `npm install`

`npm install` puts packages in a local `node_modules` folder inside your project. `pip install` installs packages globally (or into the active Python virtual environment) вЂ” there's no project-local equivalent by default.

`requirements.txt` is the Python equivalent of `package.json` вЂ” it records what packages the project needs. Generate it with:

```bash
pip freeze > requirements.txt
```

## Transcript design decisions

**Two paths to creating a transcript:**
- **Whisper** вЂ” server runs faster-whisper on the audio, produces timed segments automatically. User can edit start/end times after.
- **Manual** вЂ” user pastes text, server splits by sentence and stores segments without timing. User adds timing later if needed.

**Sentence mode** вЂ” a planned playback feature where audio plays one sentence at a time, letting the user focus on and repeat a single sentence. Requires segments aligned to sentences.

**Audio sources supported:**
- Local file upload
- Direct audio URL (podcast .mp3 etc) вЂ” downloaded with Node `fetch`
- YouTube URL вЂ” audio extracted with `yt-dlp`, then transcribed
