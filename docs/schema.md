# Database Schema

## Overview

The app uses a relational database (SQLite locally, PostgreSQL when hosted). Below are all tables, their fields, and how they relate to each other.

---

## Entity Relationships

```
User
 ├── has many Courses (filtered by active_language)
 │    └── Course has many Lessons
 │         └── Lesson has one Transcript
 │              └── Transcript has many TranscriptSegments
 ├── has many VocabLists
 └── has many Words (each optionally linked to a Lesson)
      └── Word belongs to many VocabLists (via WordVocabLists)
```

---

## Tables

### Users
Stores account information and the user's language preferences.

| Column          | Type      | Notes                                      |
|-----------------|-----------|--------------------------------------------|
| id              | INTEGER   | Primary key, auto-increment                |
| email           | TEXT      | Unique, used for login                     |
| password_hash   | TEXT      | Never store plain text passwords           |
| source_language | TEXT      | Native/translation language (e.g. `en`, `es`, `fr`) |
| active_language | TEXT      | Language currently being studied (e.g. `de`, `hu`) |
| created_at      | DATETIME  | Auto-set on creation                       |

---

### Courses
Groups lessons together under a shared theme. Tied to one target language.

| Column      | Type     | Notes                                         |
|-------------|----------|-----------------------------------------------|
| id          | INTEGER  | Primary key                                   |
| user_id     | INTEGER  | FK → Users                                    |
| title       | TEXT     |                                               |
| language    | TEXT     | Target language for this course (e.g. `de`)   |
| theme       | TEXT     | e.g. `culture`, `news`, `literature`, `travel` |
| description | TEXT     | Optional                                      |
| created_at  | DATETIME |                                               |

---

### Lessons
Individual audio lessons within a course.

| Column          | Type     | Notes                                                        |
|-----------------|----------|--------------------------------------------------------------|
| id              | INTEGER  | Primary key                                                  |
| course_id       | INTEGER  | FK → Courses                                                 |
| title           | TEXT     |                                                              |
| type            | TEXT     | `podcast`, `news_report`, `poem`, `novel`, `song`, etc.      |
| audio_url       | TEXT     | URL of the source audio (nullable if file uploaded)          |
| audio_file_path | TEXT     | Local/server path to audio file (nullable if URL used)       |
| duration        | INTEGER  | Length in seconds                                            |
| created_at      | DATETIME |                                                              |

---

### Transcripts
One transcript per lesson, generated by Whisper.

| Column     | Type     | Notes                        |
|------------|----------|------------------------------|
| id         | INTEGER  | Primary key                  |
| lesson_id  | INTEGER  | FK → Lessons (unique/one-to-one) |
| created_at | DATETIME |                              |

---

### TranscriptSegments
The individual timestamped lines of a transcript. Kept separate from Transcripts so playback sync can query efficiently by time.

| Column          | Type     | Notes                                      |
|-----------------|----------|--------------------------------------------|
| id              | INTEGER  | Primary key                                |
| transcript_id   | INTEGER  | FK → Transcripts                           |
| sequence_order  | INTEGER  | Position in the transcript (1, 2, 3, ...)  |
| start_time      | REAL     | Start time in seconds (e.g. `12.4`)        |
| end_time        | REAL     | End time in seconds                        |
| text            | TEXT     | The transcribed text for this segment      |

---

### VocabLists
Named collections of saved words belonging to a user.

| Column     | Type     | Notes                                        |
|------------|----------|----------------------------------------------|
| id         | INTEGER  | Primary key                                  |
| user_id    | INTEGER  | FK → Users                                   |
| name       | TEXT     | e.g. `Week 1`, `Travel words`, `B2 prep`     |
| language   | TEXT     | Target language of words in this list        |
| created_at | DATETIME |                                              |

---

### Words
Individual vocabulary words saved by the user while reading a lesson.

| Column          | Type     | Notes                                                              |
|-----------------|----------|--------------------------------------------------------------------|
| id              | INTEGER  | Primary key                                                        |
| user_id         | INTEGER  | FK → Users                                                         |
| lesson_id       | INTEGER  | FK → Lessons, nullable (where the word was encountered)            |
| word            | TEXT     | The word as it appears in the target language                      |
| language        | TEXT     | Target language of the word (e.g. `de`)                            |
| translation     | TEXT     | Translation in the user's source language                          |
| source_language | TEXT     | Language the translation is in (e.g. `en`) — stored explicitly so it remains correct if the user later changes their source language |
| context         | TEXT     | The full sentence the word appeared in                             |
| status          | TEXT     | `unknown`, `recognized`, `familiar`, `learned`                     |
| saved_at        | DATETIME |                                                                    |

---

### WordVocabLists
Junction table linking words to vocab lists. A word can belong to many lists.

| Column        | Type    | Notes              |
|---------------|---------|--------------------|
| word_id       | INTEGER | FK → Words         |
| vocab_list_id | INTEGER | FK → VocabLists    |

---

## Language Codes

We use [ISO 639-1](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes) two-letter codes throughout:

| Code | Language   |
|------|------------|
| `en` | English    |
| `de` | German     |
| `hu` | Hungarian  |
| `es` | Spanish    |
| `fr` | French     |
